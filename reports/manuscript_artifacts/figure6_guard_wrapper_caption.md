## Figure: guard wrapper schematic — caption (plain text)

Optional guard wrapper (evict_value_v1_guarded). On full-cache misses the base learned policy scores residents. A lightweight early-return detector tracks whether the base victim is quickly re-requested; suspicious events in a bounded time window can trigger a fixed-duration fallback to lru or marker. Both policies are stepped each request for consistent shadow state (see src/lafc/policies/guard_wrapper.py). This is an empirical control layer described in-repository; it is not asserted as a competitive robustness theorem.
